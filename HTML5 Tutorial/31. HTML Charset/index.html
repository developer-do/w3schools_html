문자 인코딩이란 무엇입니까?

ASCII는 첫 번째 문자 인코딩 표준이었습니다.(문자 집합이라고도 함) ASCII는 숫자(0-9), 영문자 (AZ) 그리고 ! $ + - ( ) @ < >와 같은 특수 문자와 같이 인터넷에서 사용할 수 있는 128개의 영숫자를 정의했습니다.

ANSI (Windows-1252)는 원래의 Windows 문자 집합으로 256가지 문자 코드를 지원합니다.

\

ISO-8859-1은 HTML 4의 기본문자 집합입니다. 이 문자 집합은 256개의 다른 문자 코드도 지원합니다.

ANSI와 ISO-8859-1은 매우 제한적 이었기 때문에 HTML 4도 UTF-8을 지원했습니다.

UTF-8(유니코드)은 세계의 거의 모든 문자와 기호를 포함합니다.

HTML5의 기본문자 인코딩은 UTF-8입니다.

HTML charset 특성

HTML 페이지를 올바르게 표시하려면 웹 브라우저가 페이지에서 사용 된 문자 집합을 알아야 합니다.

< meta >태그에 지정됩니다.

HTML4 인 경우

< meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1">

HTML5 인 경우

< meta charset="UTF-8" >

브라우저가 웹 페이지에서 ISO-8859-1을 감지하면 ANSI는 32개의 추가문자를 제외하고 ISO-8859-1과 동일 하므로 ANSI로 기본설정 됩니다.

ASCII 문자 세트

ASCII는 제어 문자에 0 에서 31 (및 127) 사이의 값을 사용합니다.

ASCII는 문자, 숫자 및 기호에 32에서 126사이의 값을 사용합니다.

ASCII는 128에서 255까지의 값을 사용하지 않습니다.

ANSI 문자집합 (Windows-1252)

ANSI는 0에서 127 사이의 값에 대해 ASCII와 동일합니다.

ANSI는 128에서 159 사이의 값에 대해 독점적인 문자 집합을 사용합니다.

ANSI는 160에서 255 사이의 값에 대해 UTF-8과 동일합니다.

ISO-8859-1 문자 세트

8859-1은 0에서 127 사이의 값에 대해 ASCII와 동일합니다.

8859-1은 128에서 159 사이의 값을 사용하지 않습니다.

8859-1은 160에서 255 사이의 값에 대해 UTF-8과 동일합니다.

UTF-8 문자 세트

UTF-8은 0에서 127 사이의 값에 대해 ASCII와 동일합니다.

UTF-8은 128에서 159 사이의 값을 사용하지 않습니다.

UTF-8은 160에서 255 사이의 값에 대해 ANSI및 8859-1과 동일합니다.

UTF-8은 값이 256에서 10000개 이상의 다른 문자까지 계속됩니다.

@charset CSS 규칙

CSS @charset 규칙을 사용하여 스타일 시트에 사용된 문자 인코딩을 지정할 수 있습니다.

ex)

스타일 시트의 인코딩을 유니코드 UTF-8로 설정합니다.

@charset "UTF-8";
